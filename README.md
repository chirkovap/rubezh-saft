# САФТ "Рубеж" — Система Анализа и Фильтрации Трафика

Программный комплекс САФТ "Рубеж" представляет собой высокопроизводительную систему защиты от DDoS-атак, реализованную на основе технологий XDP/eBPF. Фильтрация пакетов выполняется на уровне драйвера сетевого адаптера — до передачи данных в сетевой стек операционной системы. Комплекс обеспечивает динамическое управление параметрами защиты без перекомпиляции ядерной программы.


## 1. Назначение и область применения

Программный комплекс предназначен для защиты серверной инфраструктуры от атак типа "отказ в обслуживании" (DDoS) на сетевом уровне. Область применения:

- защита веб-серверов и сервисов от объёмных сетевых атак (SYN flood, UDP flood, ICMP flood);
- ограничение скорости передачи пакетов от отдельных IP-адресов по протоколу и типу трафика;
- блокировка и разблокировка IP-адресов в режиме реального времени;
- мониторинг сетевого трафика и ведение журналов событий безопасности.


## 2. Основные возможности

- Фильтрация пакетов на уровне драйвера NIC средствами XDP/eBPF
- Динамическая конфигурация через BPF-карты без перезагрузки и перекомпиляции
- Раздельные лимиты трафика для TCP SYN, UDP и ICMP пакетов
- Управление чёрным и белым списками IP-адресов (поддержка CIDR)
- Статистика трафика в реальном времени: счётчики пакетов, байт, процент блокировок
- Веб-панель управления с поддержкой русского и английского языков
- REST API для интеграции с внешними системами мониторинга
- Журналирование событий безопасности и захват пакетов
- Управление через интерфейс командной строки (CLI)
- Автоматическая синхронизация конфигурации YAML с BPF-картами ядра

## 4. Установка

### Автоматическая установка

```bash
# Клонировать репозиторий
git clone https://github.com/chirkovap/rubezh-saft.git
cd rubezh-saft

# Запустить автоматическую установку
# Скрипт компилирует XDP-программу, устанавливает зависимости,
# настраивает systemd-сервис и генерирует ключи безопасности
sudo ./scripts/install.sh
```

После завершения установки веб-панель управления доступна по адресу:

```
http://<IP-адрес-сервера>:8080
```

### Зависимости, устанавливаемые скриптом

- Системные пакеты: clang, llvm, libbpf-dev, linux-headers, bpftool
- Python-пакеты: flask, pyyaml, requests, click (согласно requirements.txt)
- Systemd-сервис: rubezh-saft.service

## 6. Конфигурация

### Механизм динамической конфигурации

Программный комплекс использует BPF-карты (`config_map`, `blacklist`, `whitelist`, `rate_limit`) для хранения параметров в ядре. При изменении файла `/etc/rubezh-saft/config.yaml` модуль ConfigSync автоматически синхронизирует значения с BPF-картами при перезапуске демона. Перекомпиляция ядерной программы не требуется.

### Настройка лимитов трафика

Файл конфигурации: `/etc/rubezh-saft/config.yaml`

```yaml
protection:
  enabled: true

  # Лимиты трафика: пакетов в секунду на один IP-адрес
  syn_rate: 1000      # TCP SYN пакеты
  udp_rate: 500       # UDP пакеты
  icmp_rate: 100      # ICMP пакеты

  # Допустимый кратковременный всплеск
  syn_burst: 2000
  udp_burst: 1000
  icmp_burst: 200
```

Применить изменения:

```bash
sudo systemctl restart rubezh-saft

# Проверить синхронизацию
sudo journalctl -u rubezh-saft -n 20 | grep -i sync
```

### Управление белым списком

Добавьте управляющие IP-адреса в белый список до запуска сервиса, чтобы исключить потерю доступа:

```yaml
whitelist_ips:
  - 127.0.0.1           # Локальный хост
  - 192.168.0.0/16      # Локальная сеть
  - 10.0.0.0/8          # Сеть VPN
  - 203.0.113.10        # Адрес управляющей станции
```

Поддерживается CIDR-нотация для подсетей.

### Сетевой интерфейс

```yaml
network:
  interface: ens33      # Указать имя сетевого интерфейса
  xdp_mode: xdpgeneric  # xdpgeneric — универсальный режим; xdpdrv — нативный
```

Получить список доступных интерфейсов: `ip link show`


## 7. Управление системой

### Управление сервисом

```bash
sudo systemctl status rubezh-saft
sudo systemctl start rubezh-saft
sudo systemctl stop rubezh-saft
sudo systemctl restart rubezh-saft

# Просмотр журнала в реальном времени
sudo journalctl -u rubezh-saft -f

# Последние 50 записей журнала
sudo journalctl -u rubezh-saft -n 50
```

### Команды CLI

```bash
cd /opt/rubezh-saft

sudo python3 cli.py status               # Статус системы и статистика
sudo python3 cli.py block 1.2.3.4        # Заблокировать IP-адрес
sudo python3 cli.py unblock 1.2.3.4      # Разблокировать IP-адрес
sudo python3 cli.py list                 # Список заблокированных адресов
sudo python3 cli.py export -o stats.json # Экспорт статистики в JSON
```

### Веб API

```bash
# Статус системы
curl http://localhost:8080/api/status

# Последние события
curl http://localhost:8080/api/events?limit=20

# Журнал пакетов
curl http://localhost:8080/api/packets?limit=100

# Блокировка IP-адреса
curl -X POST http://localhost:8080/api/block \
  -H "Content-Type: application/json" \
  -d '{"ip": "1.2.3.4", "reason": "атака"}'

# Разблокировка IP-адреса
curl -X POST http://localhost:8080/api/unblock \
  -H "Content-Type: application/json" \
  -d '{"ip": "1.2.3.4"}'
```


## 8. Веб-интерфейс

Веб-панель управления доступна по адресу `http://<IP-адрес-сервера>:8080`.

Функциональные разделы панели:

- **Обзор**: статистика трафика в реальном времени — общее число пакетов, процент блокировок, пропускная способность
- **Журнал событий**: обнаруженные атаки, выполненные блокировки, системные события
- **Журнал пакетов**: детализированный захват пакетов с возможностью фильтрации по IP и протоколу
- **Управление**: блокировка и разблокировка IP-адресов через форму ввода

Панель поддерживает переключение языка (русский / английский) и выбор цветовой темы (светлая / тёмная). Данные обновляются автоматически каждые 2 секунды.


## 9. Архитектура системы

### Ключевые компоненты

**XDP-фильтр** (`bpf/xdp_filter.c`)

Программа уровня ядра, скомпилированная в байт-код eBPF. Выполняет обработку каждого входящего пакета: разбор заголовков Ethernet/IP, проверку белого и чёрного списков, применение лимитов трафика. Принимает решение XDP_PASS или XDP_DROP до передачи пакета в сетевой стек.

**ConfigSync** (`python/config_sync.py`)

Модуль синхронизации конфигурации. Считывает значения из `config.yaml` и записывает их в BPF-карты ядра посредством вызовов `bpftool`. Обеспечивает изменение параметров защиты без перекомпиляции.

**XDPManager** (`python/xdpmanager.py`)

Центральный оркестратор пользовательского пространства. Управляет загрузкой и выгрузкой XDP-программы, делегирует задачи ConfigSync, EventLogger, PacketLogger, AttackDetector.

**Демон** (`daemon.py`)

Точка входа. Запускает XDPManager, Flask-сервер и модуль обнаружения атак AttackDetector в параллельных потоках.

**Веб-панель** (`web/app.py`, `web/templates/dashboard.html`)

Flask-приложение, предоставляющее REST API и однофайловый HTML/JS/CSS дашборд.

### Поток обработки пакета

```
Входящий пакет
    -> XDP-фильтр (ядро)
        -> Разбор Ethernet + IP
        -> config_map[CFG_ENABLED] == 0 -> XDP_PASS
        -> whitelist[src_ip] -> XDP_PASS
        -> blacklist[src_ip] -> XDP_DROP
        -> rate_limit[src_ip][protocol] превышен -> XDP_DROP
        -> XDP_PASS
    -> stats_map (обновление счётчиков)
```


## 10. Устранение неисправностей

### Потеря доступа по SSH после запуска

```bash
# Отключить XDP через консоль виртуальной машины (VMware/VirtualBox)
sudo ip link set dev ens33 xdp off
sudo systemctl stop rubezh-saft

# Добавить управляющий IP в белый список
sudo nano /etc/rubezh-saft/config.yaml

sudo systemctl start rubezh-saft
```

### Изменения конфигурации не применяются

```bash
# Проверить синхронизацию в журнале
sudo journalctl -u rubezh-saft -n 30 | grep -i sync

# Проверить корректность YAML
python3 -c "import yaml; yaml.safe_load(open('/etc/rubezh-saft/config.yaml'))"
```

### Высокий процент блокировок

```bash
# Проверить текущую статистику и активные события
curl http://localhost:8080/api/status
curl http://localhost:8080/api/events

# Увеличить лимиты в конфигурации, если блокируется легитимный трафик
sudo nano /etc/rubezh-saft/config.yaml
```

### XDP-программа не загружается

```bash
# Проверить наличие скомпилированного объекта
ls -la /usr/lib/rubezh-saft/xdp_filter.o

# Перекомпилировать при необходимости
cd /opt/rubezh-saft/bpf
sudo make clean && sudo make && sudo make install

# Убедиться, что используется режим xdpgeneric
sudo nano /etc/rubezh-saft/config.yaml
```

### Веб-панель недоступна

```bash
# Проверить, что порт 8080 прослушивается
sudo netstat -tlnp | grep 8080

# Открыть порт в брандмауэре при необходимости
sudo ufw allow 8080
```


## 11. Частые вопросы

**Требуется ли перекомпиляция при изменении config.yaml?**
Нет. Модуль ConfigSync записывает новые значения напрямую в BPF-карты ядра. Достаточно перезапустить сервис: `sudo systemctl restart rubezh-saft`.

**Почему производительность снижена при использовании xdpgeneric?**
Режим `xdpgeneric` реализован в сетевом стеке ядра и не требует поддержки драйвером. Нативный режим `xdpdrv` быстрее, но поддерживается не всеми драйверами сетевых адаптеров.

**Допустимо ли использование в промышленной эксплуатации?**
Программный комплекс функционально пригоден к развёртыванию. Перед вводом в эксплуатацию рекомендуется провести тестирование лимитов трафика в условиях, соответствующих целевой среде, а также устранить известные уязвимости (см. раздел "Известные проблемы" в CLAUDE.md).

**Как добавить целую подсеть в белый список?**
Используйте CIDR-нотацию: `192.168.0.0/24` или `10.0.0.0/8`.

**Защищает ли комплекс от атак прикладного уровня (L7)?**
Нет. Комплекс обеспечивает защиту на сетевом уровне (L3/L4) от объёмных атак. Для защиты от атак прикладного уровня требуются дополнительные средства.
